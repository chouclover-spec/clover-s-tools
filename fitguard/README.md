# FitGuard - 游戏背景图多分辨率适配预览工具

## 背景
游戏 UI 背景图需要适配多种屏幕比例（4:3 / 16:9 / 21:9 / 32:9）。
设计师上传背景图后，可快速预览各分辨率下的裁切效果，确保核心内容不被裁掉。

## 制作参数规范
- **背景图设计分辨率**：2720×1440
  - 两侧各有 80px 扩充区域（共 160px），用于 21:9 / 32:9 超宽屏的渐变过渡
- **UI 设计分辨率**：2560×1440（16:9）
- **核心安全区**：1920×1080（居中）
  - 所有重要视觉内容必须在此区域内
  - 任何比例裁切都不能丢失此区域

## 需要适配的屏幕比例

| 比例 | 目标分辨率 | 说明 |
|------|-----------|------|
| 4:3 | 1920×1440 | 从中心向两侧裁切 |
| 16:9 | 2560×1440 | 从中心向两侧裁切 |
| 21:9 | 3440×1440 | 左右各裁切 80px，上下不裁切 |
| 32:9 | 3840×1080 | 在 21:9 基础上保留左右 80px，裁切上下至 1080 高度 |

## 适配逻辑

以图片中心为锚点：

- **4:3**（1920×1440）：可视区域从中心向两侧裁切至 1920px 宽，高度保持 1440px
- **16:9**（2560×1440）：可视区域从中心向两侧裁切至 2560px 宽，高度保持 1440px
- **21:9**（3440×1440）：左右各裁切 80px（即使用 2560px 内容宽度），按目标比例计算高度
- **32:9**（3840×1080）：
  1. 从背景图 2720px 中取完整宽度
  2. 上下居中裁切至 1080px 高度
  3. 显示画布为 3840px 宽，图片居中（两侧各多出 560px 空白区域）
  4. 在图片**上方**叠加左右渐变遮罩：
     - 左遮罩：从 x=0 左对齐，覆盖 560px 空白区 + 80px 渐变过渡
     - 右遮罩：从 x=3840 右对齐，覆盖 560px 空白区 + 80px 渐变过渡
     - 遮罩不压缩，保持原始比例

## 功能说明

### 总览页（首页）
- 拖拽或点击上传一张背景图（支持 PNG / JPG / WebP）
- 上传后在 2×2 网格中同时展示 4 个比例的预览缩略图
- 每个缩略图显示比例标签和实际分辨率
- 自适应屏幕高度，保证一屏内看到全部四张图

### 叠加模式
- 在原图上叠加所选比例的裁切框线，每个比例用不同颜色
- 提供 4:3 / 16:9 / 21:9 / 32:9 勾选项，可自由组合显示
- 默认勾选并预览 4:3

### 工具栏
- **总览** / **叠加** 模式切换
- **换图** 按钮（右侧对齐）重新上传图片

## 分发方式
- `FitGuard.html`：单文件版本，所有 CSS / JS / 遮罩 / 图标全部内联，双击即用
- 原始源文件（`index.html` / `style.CSS` / `script.js`）保留用于开发维护

## 技术实现
- 纯前端（HTML + CSS + JS），无需后端
- Canvas API 处理图片裁切与遮罩渲染
- 32:9 遮罩通过 Canvas 动态生成渐变图（无需外部 PNG 文件）
- 响应式 CSS Grid 布局
